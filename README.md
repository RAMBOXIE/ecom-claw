# 🦞 Clawbot for Ecommerce

> 面向电商小团队的全生命周期 AI 经营助手。

---

## 项目定位

**Clawbot 不是"会模拟人点击后台的万能机器人"。**

它是一套把电商运营工作抽象为**可理解、可配置、可执行、可审计、可回滚**的任务与 SOP 系统。

### 目标用户

**3~20 人的电商小团队**，包括：
- 独立站品牌团队
- 跨境卖家团队
- 国内内容/社群驱动型电商团队

### 核心价值

| 问题 | Clawbot 的解法 |
|------|---------------|
| 上新重复劳动多 | Launch Bot 自动生成商品页草稿 + SEO/GEO 补全 |
| 日报周报耗时 | Ops Bot 自动生成 + 推送 |
| 社区问题无法沉淀 | Community Bot 归纳 → 回流 FAQ |
| 运营经验难复制 | SOP 模板中心 |
| 多平台数据分散 | 统一 Dashboard |
| 高风险操作无法审计 | 审批中心 + 完整操作日志 |

### 不做什么（第一阶段）

- ❌ 不做浏览器自动化主链路（RPA优先）
- ❌ 不做复杂多 Agent 自由协商
- ❌ 不自动执行高风险操作（退款/大范围改价/自动发货）
- ❌ 不做重度 ERP 深度集成
- ❌ 不做广告投放

## 核心架构

```
单主控 Agent + 多业务模块 + 确定性工作流 + 审批与审计系统
```

### 执行层优先级

1. **官方 API**（首选）
2. **CSV / 表格 / 导入导出**（次选）
3. **浏览器自动化**（兜底，不作主链路）

### 业务模块（MVP 四大模块）

| 模块 | 职责 |
|------|------|
| 🔍 Selection Bot | 选品线索、竞品分析、市场机会判断 |
| 🚀 Launch Bot | 商品页草稿、SEO/GEO 补全、文案生成 |
| 💬 Community Bot | 社区问题归纳、FAQ 回流、UGC 整理 |
| 📊 Ops Bot | 日报/周报、库存预警、页面巡检 |

---

## 电商生命周期覆盖

```
选品 → 上架与传播 → 转化优化 → 日常运营 → 社区维护 → 复盘与增长
```

| 阶段 | Clawbot 输出 |
|------|------------|
| 选品 | 候选品类、竞品摘要、用户痛点、优先级建议 |
| 上架与传播 | 商品标题、卖点、描述、FAQ、SEO 标签、GEO 内容 |
| 转化优化 | 页面缺失项识别、文案优化建议、属性完整性检查 |
| 日常运营 | 日报/周报、页面巡检、库存/价格异常预警 |
| 社区维护 | 高价值问题摘要、评论关键词、FAQ 补充建议 |
| 复盘与增长 | 自然流量内容分析、GEO/SEO 缺口、SKU 追加建议 |

## SEO 与 GEO

### SEO
让商品页、分类页在传统搜索引擎中被更好地抓取、理解、收录和排名。

### GEO（Generative Engine Optimization）
让商品、页面、品牌内容更容易被 **AI 搜索 / AI 答案 / AI 导购系统**理解、引用和推荐。

**GEO 落地到具体动作：**
- 商品层：补全属性/规格/适用场景、生成可引用 FAQ、生成对比段落
- 页面层：标题/描述/H 结构、图片 alt、评价摘要、售后事实模块
- 内容层：选购指南、对比文章、场景内容、使用说明
- 社区层：用真实用户语言补充页面表达

## 平台支持

| 平台 | 状态 | 备注 |
|------|------|------|
| Shopify | ✅ 已接入 | 订单/商品/库存/客户全覆盖 |
| WooCommerce | 🚧 v2.0 开发中 | WordPress 独立站，REST API 接入 |
| 有赞 | ⏳ 待配置 | 框架已建，需填 access_token |
| 淘宝/天猫 | 🔜 v3.0 规划中 | 需 TOP API |
| Amazon | 🔜 v3.0 规划中 | SP-API |

> **为什么 WooCommerce 和 Shopify 同优先级？**
> WooCommerce 是全球装机量最大的电商插件，WordPress 独立站占全球电商 30%+。
> 其 REST API（`/wp-json/wc/v3/`）结构与 Shopify 相近，connector 层复用成本低，
> 大部分脚本（日报/订单/库存/商品）可直接复用，只换底层 connector。

## 国内 vs 跨境

一个核心引擎 + 两套业务模板，不做两套独立系统。

| | 国内模板 | 跨境模板 |
|--|---------|---------|
| 重点 | 上新活动、评论客服、社群私域 | 多语言页面、本地化 SEO/GEO、多市场拆分 |
| 内容 | 私域触达建议、FAQ/客服摘要 | 市场筛选、多语言文案、履约/时效清晰化 |

## 操作安全规则

| 操作类型 | 规则 |
|--------|------|
| 查询 / 报告 / 预览 / 草稿 | ✅ 直接执行 |
| 改价 / 改库存 | 需审批确认 |
| 退款 / 取消订单 / 发货 | 需审批 + 日志记录 |
| 批量操作 | 先 dry-run 预览 → 审批 → 执行 |
| 大范围价格修改 | 默认禁止自动执行 |

## 配置策略

**不让用户从零配置 Bot。先选目标，再生成配置。**

### 首次引导（5分钟完成）
1. 你主要卖什么类型商品？
2. 主要在哪些渠道卖？
3. 偏国内、跨境，还是两者都有？
4. 最想优先解决什么？（选品/上新/流量/社区/巡检）
5. 哪些动作必须人工审批？

### 三套业务模板

| 模板 | 适用 | 重点功能 |
|------|------|---------|
| A - 国内内容电商 | 淘宝/抖店/小红书卖家 | 上新、评论/FAQ、私域建议、日报 |
| B - 独立站品牌 | Shopify 独立站 | 商品页生成、SEO/GEO、内容选题、页面巡检 |
| C - 跨境卖家 | 多市场跨境 | 多语言商品页、本地化、评论摘要、渠道巡检 |
